# Game Concept – UD02

## Ejercicio 1 · Parte del Game Concept

### Temática
Gestión de un restaurante caótico donde varios cocineros cooperan para atender oleadas crecientes de clientes. El foco está en la **organización**, la **cooperación** y la **optimización progresiva** del espacio mediante automatización.

**Pilares derivados de la temática:**
- Caos controlado
- Cooperación local/online
- Progresión entre días
- Mejora progresiva mediante objetos upgradeables

### Género
- **Género principal:** Gestión / Simulación
- **Subgéneros:** Party game cooperativo, Time management

### Tipo de juego
Juego de gestión cooperativa por días (niveles) donde los jugadores preparan y sirven platos a clientes dentro de un tiempo limitado.

**Key Features:**
- Hasta 4 jugadores cooperativos
- Días progresivos con dificultad creciente
- Reinicio del día al perder sin perder progreso global
- Objetos de cocina upgradeables que mejoran eficiencia

### Punto de vista
- **3D con cámara cenital/isométrica fija compartida**

### Elevator Pitch
*Un restaurante cooperativo donde el caos se reinicia cada día, pero tu progreso no.*

## Ejercicio 2 · Mockup

**Mockup elegido:** HUD principal durante la partida.

El mockup representa la interfaz visible durante el gameplay, priorizando una lectura clara sin invadir el área de juego.

**Elementos del HUD:**
- Indicador de día y temporizador en la esquina superior izquierda.
- Contador de dinero visible en todo momento.
- Barra global de satisfacción de clientes en la parte superior, que refleja el estado general del restaurante.
- Órdenes de los clientes mostradas mediante bocadillos sobre cada cliente, con icono del plato y una barra de paciencia individual.

No se muestran controles de los personajes en pantalla para mantener la interfaz limpia y evitar saturación visual.

El área central queda completamente libre para el mapa y la acción, permitiendo una visión amplia del restaurante y sus pasillos.

## Ejercicio 3 · Método MoSCoW

### Must have
- Juego cooperativo hasta 4 jugadores
- Sistema de días como niveles
- Preparación y servicio de platos
- Clientes con paciencia

### Should have
- Progresión persistente
- Sistema de mejoras de objetos (cocinas, mesas, estaciones)

### Could have

- Varias recetas
- Modificadores aleatorios de día

### Won’t have (para esta versión)
- Matchmaking automático
- Chat de voz integrado
- Personalización visual avanzada
